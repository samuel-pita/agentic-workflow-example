# Mis notas de referencia

Acá está lo que necesito a mano para armar la presentación. Léelo antes de planear;
de aquí salen las specs. Hay dos grupos:

## El harness (reusable entre presentaciones)

- **`00-brief.md`** — qué es el producto, el formato y las convenciones.
- **`01-design-system.md`** — los colores del logo, tipografías y los assets de marca.
- **`02-architecture.md`** — cómo está armado: los slides, la navegación, el `.pipe`
  y la regla de no-scroll.

Estos tres no cambian de un tema a otro. Si haces otra presentación, se quedan igual.

## El contenido (cambia por tema)

- **`03-content.md`** — el temario completo del tema. Esto es la fuente de verdad del
  contenido: secciones, slides y su copy.
- **`04-…`** — notas de dominio del tema: lo que el copy necesita para no equivocarse
  con los conceptos. (En esta presentación es `04-claude-concepts.md`.)

Para reusar la plantilla en otro tema, reemplaza este grupo y deja el harness igual.

## Atajos

Si vas a tocar **contenido**, empieza por `03`. Si es algo **visual**, mira `01` y
`02`. Para el copy conceptual, las notas de dominio (`04`).
